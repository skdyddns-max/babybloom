# 베이비블룸(BabyBloom) 기획서 v1.1

> 작성일: 2026-07-08 (v1.1 같은 날 확장: 동생 위시리스트 반영) · 작성: 용디쌤 + Claude
> 한 줄 정의: **아기 생년월일 하나만 넣으면 예방접종·원더윅스·영유아검진·언어발달 체크포인트가 자동으로 깔리는 육아 비서 달력**

## 1. 문제 정의 (동생의 실제 니즈)

실제 대화에서 나온 페인포인트 그대로:

| 동생의 말 | 해석 |
|---|---|
| "이게 달력인뎅 내가 수기로 적거든?" | 종이 육아달력(너와의열두달)은 예쁘지만 **전부 수동** |
| "도아 출생일에 맞춰서 예방접종이나 원더윅스 기간이 체크되어 있음 좋겠다" | **생일 기반 자동 일정 생성**이 핵심 니즈 |
| "마미톡이나 베이비타임 많이 쓰는데 그냥 기록용" | 기존 앱은 수유·수면 **기록**(과거) 중심, **예고**(미래)가 약함 |
| "약간 육아비서 같은 게 있음 좋겠어" | 오늘/이번 주에 뭘 챙겨야 하는지 **먼저 알려주는** 경험 |

### 기존 앱과의 차별점
- **베이비타임/마미톡**: 수유·기저귀·수면 기록(입력 노동). 베이비블룸은 **입력 1회(생일) → 평생 일정 자동**.
- **원더윅스 공식 앱**: 유료 + 원더윅스만. 베이비블룸은 접종+검진+발달을 **한 화면**에.
- **용디쌤만의 무기**: 언어재활사 17년 → 개월별 **언어발달 체크포인트 + 빨간불 신호** 탑재. 어떤 육아 달력 앱에도 없는 전문성. TalkSeed·TalkBloom으로 자연 연결되는 퍼널.

## 2. 타깃 & 사용 시나리오

- **1차 타깃**: 0~24개월 아기의 양육자(동생 = 페르소나 0호, 도아 = 첫 사용자)
- **시나리오**: 첫 실행에서 아기 이름+생년월일(+출산예정일) 입력 → 홈에서 "D+127일, 이번 주: 원더윅스 4차 도약기 진행 중 / 6개월 접종(DTaP 3차 외 4종) 2주 남음" 확인 → 접종 다녀오면 체크 → 끝.

## 3. 핵심 기능 (MVP 범위)

### F1. 원클릭 온보딩
- 입력: 아기 이름, 생년월일(필수), 출산예정일(선택 — 원더윅스 교정 계산용), 로타 백신 종류(로타텍 3회/로타릭스 2회)
- 저장: localStorage (서버·로그인 없음 → 즉시 배포 가능)

### F2. 오늘(홈) — "육아비서" 화면
- D+일수, 생후 주수·개월수, (조산 시) 교정 주수
- **이번 주 브리핑 카드**: 진행 중 원더윅스 도약기 / 30일 내 다가오는 접종·검진 / 현재 개월의 언어발달 체크포인트
- 지난 접종 미체크 시 "놓친 접종 확인" 알림 카드

### F3. 접종·검진 자동 캘린더
- 질병관리청 표준예방접종일정표 기준 **전체 접종을 날짜로 자동 변환** (예: 생후 2개월 → 2026-09-08)
- 영유아 건강검진 8차(14~35일 ~ 66~71개월) + 구강검진 자동 배치
- 완료 체크(탭 1번) → 완료/예정/지연 상태 구분
- 시기 범위형(12~15개월 등)은 범위로 표시

### F4. 원더윅스 트래커
- 출산예정일(없으면 생일) 기준 도약기 1~10차 자동 계산
- 현재 단계 하이라이트 + 진행바, 각 도약기 이름·아기 변화·부모 대처팁
- "±1~2주 개인차" 고지 문구 상시 노출

### F5. 언어발달 체크포인트 (시그니처)
- 개월 구간별 "이 시기 아기는" + 체크리스트(부모가 직접 체크)
- **빨간불 신호**: 12개월 옹알이 없음 / 18개월 첫 낱말 없음 / 24개월 두 낱말 조합 없음 → 전문가 상담 권고 + TalkSeed 발달체크 링크
- 감수: 용디쌤(언어재활사) 명시 → 신뢰 포인트

## 5. 기술 스택 & 아키텍처

- **MVP**: 순수 HTML/CSS/JS + localStorage, 모바일 퍼스트 (동생이 폰으로 씀)
- **배포**: GitHub Pages (skdyddns-max.github.io/babybloom) — 기존 프로젝트와 동일 파이프라인
- **구조**: `index.html` / `css/style.css` / `js/data.js`(일정 데이터) / `js/app.js`(계산·렌더)
- 데이터와 로직 분리 → 일정표 개정 시 data.js만 교체

## 6. 화면 구성 (4탭)

1. **오늘**: D+n · 브리핑 카드 3종(원더윅스/접종/발달)
2. **일정**: 전체 타임라인(접종+검진), 완료 체크
3. **원더윅스**: 10단계 카드 + 현재 단계 강조
4. **발달**: 개월별 언어발달 체크리스트 + 빨간불

## 6.5 동생 위시리스트 전체 매핑 (2026-07-08 추가)

동생이 보낸 2차 요구사항을 구현 난이도 기준으로 3분류:

### A. 바로 구현 가능 (정적 데이터+localStorage) → ✅ v1.1에서 구현 완료
| 요구 | 구현 |
|---|---|
| 똥 색깔 체크 + 이전과 비교 | 기록 탭: 7색 스와치 탭 1번 기록, 직전 기록과 자동 비교 문구, 위험색(빨강·검정·흰색)은 홈에 경고 카드 + 진료 권고 |
| 수유량 기억 + 패턴·통계 자동 | 기록 탭: 분유(ml)/모유(분)/유축/이유식(g) 기록 → 오늘 합계·최근 7일 하루평균 자동, 급감 시(평균 대비 -30%) 알림 문구 |
| 이유식으로 이어지기 + 이유식 하는 법 | 개월수 기반 이유식 단계 자동 표시(초기/중기/후기/완료기 + 형태·횟수·팁) |
| 알레르기 체크 | 새 재료 등록 → 3일 관찰 카운트 → 이상없음/반응있음 처리, 주요 알레르기 식품·꿀 금지·응급 기준 안내 |
| 개월별 놀이방법 | 발달·놀이 탭: 시기별 놀이법 4가지씩 |
| 개월별 발달단계 확인 | 발달·놀이 탭: 언어(기존) + 대근육·소근육 요약 추가 |
| 장난감 순서·단계 (무작정 안 사게) | 시기별 추천 장난감 + **"아직 사지 마세요/주의" 가이드**(보행기 비권장, 전자토이 시기 등) |

### B. AI(LLM API) 필요 → Phase 3
- AI 식단 짜기(음식 궁합), 기록 패턴 자연어 해석("요즘 수유 간격이 벌어지고 있어요"), 육아 Q&A 챗
- Claude API 키 필요(비용 발생). 서버리스 프록시(Cloudflare Workers 등) 경유 권장 — 키 노출 방지
- 그 전까지는 규칙 기반 인사이트(수유 급감 감지 등)로 "비서 느낌" 유지

### C. 외부 데이터 필요 → Phase 4
- **동네 병원 위치·영업시간**: 건강보험심사평가원 병원정보 공공 API(무료) + 예방접종 위탁의료기관 목록. 야간·휴일은 응급의료정보(E-Gen) API
- **아이 데리고 갈 만한 곳**(기저귀갈이대·유모차·수유실 여부 확인되는 카페): 이 조건까지 확인되는 공공 데이터 없음 → 수유실 위치는 공공데이터 있음. 카페 시설 정보는 커뮤니티 제보형(사용자 등록)이 현실적 — 서비스 차별점이 될 수 있는 큰 기능
- **지역 축제·행사**: 한국관광공사 TourAPI(무료, 지역·날짜 필터 가능)

## 8. 열린 결정 (사용자 확정 필요)

1. **이름**: 베이비블룸(가칭) vs 도아달력 스타일 개인화 네이밍 vs 기타
2. **원더윅스 상표**: "Wonder Weeks"는 상표 — 앱 내 표기를 "도약기(원더윅스 시기)"로 순화할지
3. **배포 시점**: 동생 베타 먼저 vs 바로 공개
